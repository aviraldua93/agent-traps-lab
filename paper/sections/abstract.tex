\begin{abstract}
Google DeepMind's ``AI Agent Traps'' taxonomy identifies six categories
of environmental attacks against autonomous AI agents, yet the original
work remains purely theoretical---no open testbed exists to empirically
measure how real agents behave when deployed into adversarial
environments. We bridge this gap with \textsc{agent-traps-lab}, an
open-source framework that operationalizes all six trap categories as
22 reproducible adversarial scenarios and evaluates them against
GPT-4o-mini (via GitHub Models) over Google's Agent-to-Agent (A2A)
protocol. Our experiment comprises 220 runs (22~scenarios $\times$
2~conditions $\times$ 5~repetitions), comparing a \emph{baseline} (no
defenses) against a \emph{hardened} configuration (seven mitigation
modules plus a security-hardened system prompt). We report trap success
rates, detection rates, Cohen's $d$ effect sizes, and
Bonferroni-corrected Wilcoxon signed-rank tests. Key findings:
(1)~baseline trap success is \textbf{45.5\%} overall, reduced to
\textbf{25.5\%} under hardened conditions---a 20 percentage-point
mitigation benefit; (2)~content injection via CSS and dynamic cloaking
resists all mitigations (100\% in both conditions), while authority
framing and cross-contamination are \emph{completely} mitigated
(100\%$\to$0\%); (3)~systemic traps show 0\% baseline success,
indicating that GPT-4o-mini naturally resists protocol-level attacks;
(4)~two \emph{regression} cases emerge where mitigations \emph{increase}
vulnerability (misleading forms 0\%$\to$100\%, ranking manipulation
80\%$\to$100\%); and (5)~the modest overall benefit suggests that
architectural defenses beyond prompt-level hardening are needed. We
release the complete testbed, experiment data, and analysis scripts.
\end{abstract}
