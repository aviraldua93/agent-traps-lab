\section{Results: Human-in-the-Loop}
\label{sec:results-hitl}

The three human-in-the-loop scenarios test whether the agent produces
biased outputs that exploit cognitive biases in human reviewers.
\Cref{tab:hitl-success} reports manipulation success rates.

\subsection{Key Findings}
\label{sec:hitl-findings}

\begin{enumerate}[nosep,leftmargin=*]
  \item \textbf{All three HitL scenarios are fully mitigated} under
    hardened conditions: 60\%$\to$0\% for cherry-picking and anchoring,
    40\%$\to$0\% for decision fatigue. Effect sizes are large
    ($d = 1.03$--$1.55$), though none reach Bonferroni significance
    at $n=5$.

  \item \textbf{Cherry-picked evidence (HL-1) and anchoring (HL-2)}
    show identical vulnerability profiles (60\% baseline, $d = 1.55$),
    suggesting a common underlying susceptibility to selective
    reporting that the \mitigation{report-auditor} module addresses.

  \item \textbf{Decision fatigue (HL-3)} has the lowest baseline
    success (40\%), consistent with the expectation that fatigue-based
    manipulation is a weaker attack vector for non-human agents.

  \item \textbf{Caveat}: These results measure whether the agent
    \emph{produces} biased output, not whether human reviewers are
    actually \emph{deceived}. A user study would be needed to
    validate the downstream impact.
\end{enumerate}
